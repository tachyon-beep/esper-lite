## XC: Simic RL soundness (PPO, GAE, recurrent rollout buffer, reward economy, normalization, action-conditioned critic)
Simic implements a factored, recurrent (LSTM) PPO with per-head causal masking, value/return normalization, and an elaborate counterfactual reward economy. The engineering is unusually careful: BPTT leakage is guarded at construction time, GAE is per-env (no interleaving), truncation-vs-terminal is handled correctly (host-training episodes are correctly treated as time-limit truncations that bootstrap), advantage re-normalization is made idempotent, and a finiteness gate fails safe. The dominant soundness concern is architectural, not a coding bug: the critic is an ACTION-CONDITIONED Q(s,op) used directly as the PPO baseline and as the GAE bootstrap, which violates the standard PPO requirement that the baseline be action-independent. For the op head the advantage subtracts a baseline that contains the op being scored (partial advantage cancellation / control-variate bias), and the bootstrap V(s') is a single Monte-Carlo Q(s',op'~beh) sample rather than E_op[Q], injecting policy-dependent variance into the value target. The reward economy is dense and multi-component; PBRS telescoping is implemented correctly, but several large non-potential dense terms (immediate fossilize bonus, terminal accuracy bonus, holding/blending penalties, prune bonuses) sit on top of PBRS, so the 'optimal-policy-invariance' guarantee applies only to the PBRS component, not the reward as a whole — and the prune bonus / drip surfaces retain residual reward-hacking exposure that the code partially mitigates with age gates. mark_terminal_with_penalty zeroes the whole episode's intermediate rewards on governor rollback, which is a deliberate but aggressive credit-assignment choice. Entropy-collapse defenses (per-head floors on AVAILABILITY masks, probability floors, scheduled penalty) are strong and a genuine positive.
  - [high] Critic is action-conditioned Q(s,op) but used as the PPO state-value baseline — baseline is not action-independent
      EV: factored_lstm.py:557-576 `_compute_value` concatenates a one-hot of `op` into the value head input; evaluate_actions uses the STORED op (factored_lstm.py:1242-1244 `stored_op = actions['op']; value = self._compute_value(lstm_out, stored_op)`). This `value` is what feeds GAE (rollout_buffer.py:509-557) and the value loss (ppo_update.py:341-353). Standard PPO requires the baseline b(s) to be independent of the action whose advantage it scores; A^op = r + gamma·V(s') - Q(s,op_taken) subtracts a baseline that already contains op_taken, so for the op head the surrogate is no longer an unbiased advantage-weighted policy gradient — part of the op's own value is cancelled out of its advantage. The 7 other heads are scored against a baseline conditioned on a DIFFERENT factor (op), which is action-independent w.r.t. them, so they are less affected; the op head is the one with the soundness problem.
      REC: Either (a) train a true state-value head V(s) (op-independent) for the PPO baseline/GAE and keep Q(s,op) only as a telemetry/aux head, or (b) if you intend an advantage-style baseline use the marginal V(s)=sum_op pi(op|s)·Q(s,op) (expectation over the CURRENT op policy) as the baseline so the op term cancels in expectation as a proper control variate. At minimum, document this as a deliberate deviation and validate with explained-variance + a V(s)-baseline ablation; the op-head policy gradient is the most likely place to see silent bias.
  - [high] GAE bootstrap V(s') uses a single behavior-sampled Q(s', op'~pi_old), not E_op[Q] — Monte-Carlo value target injects policy-variance and is off-policy after the first epoch
