## Detailed Description

### What's Wrong

The validation functions call reduction operations that require CPU sync:

```python
def _validate_action_mask(mask: torch.Tensor) -> None:
    """Validate action mask tensor."""
    if not mask.any():  # CPU sync!
        raise ValueError("All actions are masked - at least one must be valid")

def _validate_logits(logits: torch.Tensor, mask: torch.Tensor) -> None:
    """Validate logits have valid values for unmasked positions."""
    unmasked = logits[mask]
    if unmasked.numel() > 0:
        if not torch.isfinite(unmasked).all():  # CPU sync!
            ...
```

These syncs:
1. Block the GPU pipeline
2. Transfer data from GPU to CPU
3. Wait for result before continuing

### Impact

- **Medium severity**: Called on every action sample if validation enabled
- **Performance**: Each sync can add 50-200μs latency
- **Cumulative**: With 128 envs, thousands of syncs per rollout

### Mitigation

The code already has a `MaskedCategorical.validate` class variable to disable validation:

```python
class MaskedCategorical:
    validate: ClassVar[bool] = True  # Disable for production
```

The issue is that this is a class variable (affects all instances) and requires code modification to disable.

---

## Recommended Fix

Options:

1. **Environment variable control** (recommended):
```python
import os
MaskedCategorical.validate = os.environ.get("ESPER_VALIDATE_MASKS", "0") == "1"
```

2. **Instance-level control**:
```python
def __init__(self, logits: torch.Tensor, mask: torch.Tensor, validate: bool = True):
    if validate and MaskedCategorical.validate:
        self._validate(logits, mask)
```

3. **torch.compile decorator** to avoid sync (complex):
```python
@torch.compiler.disable
def _validate_action_mask(mask: torch.Tensor) -> None:
    # Already has this decorator, but sync still occurs
```
