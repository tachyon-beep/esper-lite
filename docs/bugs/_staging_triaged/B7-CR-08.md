## Detailed Description

### What's Wrong

```python
@dataclass
class AnomalyReport:
    anomalies: list[AnomalyType] = field(default_factory=list)
    details: dict[AnomalyType, str] = field(default_factory=dict)

    def add(self, anomaly_type: AnomalyType, detail: str) -> None:
        self.anomalies.append(anomaly_type)
        self.details[anomaly_type] = detail  # Overwrites if key exists!
```

If the same anomaly type is added twice:
1. Both are added to `anomalies` list (duplicates allowed)
2. But only the last `detail` is kept in `details` dict

Example:
```python
report.add(AnomalyType.RATIO_EXPLOSION, "ratio=1.5 in head=0")
report.add(AnomalyType.RATIO_EXPLOSION, "ratio=2.0 in head=3")
# anomalies = [RATIO_EXPLOSION, RATIO_EXPLOSION]
# details = {RATIO_EXPLOSION: "ratio=2.0 in head=3"}  # First detail lost!
```

---

## Recommended Fix

**Option 1 - Use list[str] for details:**
```python
details: dict[AnomalyType, list[str]] = field(default_factory=dict)

def add(self, anomaly_type: AnomalyType, detail: str) -> None:
    self.anomalies.append(anomaly_type)
    if anomaly_type not in self.details:
        self.details[anomaly_type] = []
    self.details[anomaly_type].append(detail)
```

**Option 2 - Detect and warn on duplicates:**
```python
def add(self, anomaly_type: AnomalyType, detail: str) -> None:
    if anomaly_type in self.details:
        _logger.warning("Overwriting detail for %s", anomaly_type)
    # ...
```
