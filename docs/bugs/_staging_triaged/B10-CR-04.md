## Detailed Description

### What's Wrong

The registry provides:
- `register_policy(name)` decorator to add policies
- `get_policy(name)` to retrieve policies
- `list_policies()` to see all registered policies
- `clear_registry()` to remove ALL policies (for testing)

Missing: `unregister_policy(name)` to remove a single policy.

### Impact

- **Low severity**: Edge case for development workflows
- **Hot reload**: Cannot unregister and re-register during module reload
- **Testing**: Must clear entire registry to test registration behavior

### Use Case

During development, if a policy implementation changes significantly:
```python
# Currently impossible without clearing everything
unregister_policy("lstm")  # Would be nice
register_policy("lstm")(NewLSTMBundle)  # Re-register updated version
```

---

## Recommended Fix

Add unregister function:

```python
def unregister_policy(name: str) -> bool:
    """Remove a policy from the registry.

    Args:
        name: Policy name to remove

    Returns:
        True if policy was removed, False if it wasn't registered.
    """
    if name in _POLICY_REGISTRY:
        del _POLICY_REGISTRY[name]
        return True
    return False
```
