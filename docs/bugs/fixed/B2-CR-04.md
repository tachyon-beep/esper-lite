## Detailed Description

### What's Wrong

`AlphaController.from_dict()` uses `.get(key, default)` pattern for deserialization. Per CLAUDE.md, this is a prohibited defensive programming pattern:

> "Do not use .get()... to suppress errors from... malformed data... When code fails, fix the actual cause... If code would fail without a defensive pattern, that failure is a bug to fix, not a symptom to suppress."

### Code Evidence

```python
# /home/john/esper-lite/src/esper/kasmina/alpha_controller.py:147-155

return cls(
    alpha=float(data.get("alpha", 0.0)),  # Should fail if missing
    alpha_start=float(data.get("alpha_start", 0.0)),
    alpha_target=float(data.get("alpha_target", 1.0)),
    step=int(data.get("step", 0)),
    total_steps=int(data.get("total_steps", 1)),
    mode=AlphaMode(data.get("mode", AlphaMode.HOLD.value)),
    curve=AlphaCurve(data.get("curve", AlphaCurve.LINEAR.value)),
)
```

### Comparison

`SeedState.from_dict()` in slot.py correctly uses direct access with explicit error handling.

---

## Recommended Fix

Align with the project pattern - direct access, fail fast:

```python
@classmethod
def from_dict(cls, data: dict[str, Any]) -> "AlphaController":
    """Deserialize from checkpoint dict.

    Raises:
        KeyError: If required fields are missing
        ValueError: If field values are invalid
    """
    return cls(
        alpha=float(data["alpha"]),
        alpha_start=float(data["alpha_start"]),
        alpha_target=float(data["alpha_target"]),
        step=int(data["step"]),
        total_steps=int(data["total_steps"]),
        mode=AlphaMode(data["mode"]),
        curve=AlphaCurve(data["curve"]),
    )
```
