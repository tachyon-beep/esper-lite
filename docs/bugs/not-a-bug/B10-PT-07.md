## Detailed Description

### What's Wrong

```python
def slot_id_to_index(slot_id: SlotId, slot_config: SlotConfig) -> int:
    """Convert a SlotId to its index in the slot config."""
    from esper.leyline import SlotConfig  # Import inside function

    # ... function body
```

While Python caches imports (so repeated imports are fast), this pattern:
1. Adds a small overhead on every function call
2. Makes dependencies harder to trace
3. Is typically used to avoid circular imports

### Impact

- **Minimal**: Python import caching makes overhead negligible
- **Style**: Non-standard import location
- **Readability**: Dependencies not visible at module level

---

## Recommended Fix

Move import to module level if no circular import:

```python
# At module level
from esper.leyline import SlotConfig

def slot_id_to_index(slot_id: SlotId, slot_config: SlotConfig) -> int:
    """Convert a SlotId to its index in the slot config."""
    # ... function body
```

If circular import exists, consider:
1. Import in `if TYPE_CHECKING:` for type hints
2. Restructure modules to break cycle
3. Keep local import with comment explaining why
