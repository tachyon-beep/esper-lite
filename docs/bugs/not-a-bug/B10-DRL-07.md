## Detailed Description

### What's Wrong

From the protocol definition:

```python
@property
def hidden_dim(self) -> int:
    """Hidden state dimension.

    For non-recurrent policies, return 0 or raise NotImplementedError.
    """
    ...
```

This "or" creates ambiguity:
1. Should callers check for 0 or catch exceptions?
2. Different implementations may choose differently
3. Code using this property must handle both cases

### Impact

- **Minimal**: Style/consistency issue
- **Code complexity**: Callers must handle both cases
- **Future maintenance**: Inconsistent implementations

---

## Recommended Fix

Pick one behavior and document it clearly:

**Option A (Preferred - return 0):**
```python
@property
def hidden_dim(self) -> int:
    """Hidden state dimension.

    Returns:
        The hidden state dimension. For non-recurrent policies,
        returns 0 (no hidden state needed).
    """
    ...
```

**Option B (NotImplementedError):**
```python
@property
def hidden_dim(self) -> int:
    """Hidden state dimension.

    Returns:
        The hidden state dimension.

    Raises:
        NotImplementedError: If policy is not recurrent.
    """
    ...
```

Option A is preferred because:
1. Return values are easier to handle than exceptions
2. 0 is a meaningful value (no hidden state)
3. Avoids try/except patterns in calling code
