## Detailed Description

### What's Wrong

```python
def emit_last_action(...) -> dict[str, Any]:
    """Emits action event AND returns dict."""
    # ... builds payload ...
    hub.emit(event)  # Side effect
    return {          # Also returns value
        "action_type": action_type,
        "slot_target": slot_target,
        # ...
    }
```

Issues:
1. **Dual responsibility:** Violates single responsibility principle
2. **Confusing API:** Is the return value the primary purpose or a convenience?
3. **Potential legacy:** Comment suggests "for backwards compat"
4. **Semantic mismatch:** `emit_*` functions typically return None

Per No Legacy Code Policy: If the return value was for backwards compatibility, it should be removed and callers updated.

---

## Recommended Fix

**Option 1 - Pure emission (preferred):**
```python
def emit_last_action(...) -> None:
    """Emits action telemetry event. Returns None."""
    # ... builds payload ...
    hub.emit(event)
    # No return
```

**Option 2 - Separate functions:**
```python
def build_action_dict(...) -> dict[str, Any]:
    """Build action data without emitting."""
    return {...}

def emit_last_action(...) -> None:
    """Emit action telemetry."""
    data = build_action_dict(...)
    hub.emit(Event(**data))
```
