## Detailed Description

### What's Wrong

```python
# Lines 67-69: Vanilla EMA without bias correction
self.ema_norm = self.momentum * self.ema_norm + (1 - self.momentum) * grad_norm
self.ema_health = self.momentum * self.ema_health + (1 - self.momentum) * grad_health
```

With momentum=0.99, the EMA is heavily biased toward the initial value (0.0) during early updates:

| Update # | Bias Factor (0.99^t) | Effective Weight of Observations |
|----------|---------------------|----------------------------------|
| 1 | 0.99 | 1% |
| 10 | 0.90 | 10% |
| 100 | 0.37 | 63% |
| 500 | 0.007 | 99% |

This means drift detection during the first ~100-500 updates will be unreliable.

**Standard fix (Adam optimizer uses this):**
```python
corrected_ema = ema / (1 - momentum ** update_count)
```

---

## Recommended Fix

Add optional bias correction:

```python
def update(self, grad_norm: float, grad_health: float) -> None:
    self._update_count += 1

    self.ema_norm = self.momentum * self.ema_norm + (1 - self.momentum) * grad_norm
    self.ema_health = self.momentum * self.ema_health + (1 - self.momentum) * grad_health

    # Bias correction for early updates
    if self.use_bias_correction:
        bias_correction = 1 - self.momentum ** self._update_count
        self._corrected_norm = self.ema_norm / bias_correction
        self._corrected_health = self.ema_health / bias_correction
```

### Reference

Adam optimizer (Kingma & Ba, 2014) uses bias correction for the same reason:
```
m_t = beta1 * m_{t-1} + (1 - beta1) * g_t
m_t_corrected = m_t / (1 - beta1^t)
```
