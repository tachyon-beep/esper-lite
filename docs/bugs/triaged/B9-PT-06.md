## Detailed Description

### What's Wrong

```python
# Lines 87-98 (constructor parameters)
def __init__(
    self,
    obs_dim: int,
    slot_config: SlotConfig | None = None,
    num_slots: int | None = None,        # Can override slot_config
    num_blueprints: int | None = None,   # Can override slot_config
    num_styles: int | None = None,       # ...etc
    ...
):

# Lines 109-122 (fallback logic)
if num_slots is None:
    num_slots = slot_config.max_slots if slot_config else 4
if num_blueprints is None:
    num_blueprints = slot_config.num_blueprints if slot_config else 13
# ... more fallbacks
```

This creates two paths to configure the same thing:
1. Pass `slot_config` and let it determine sizes
2. Pass explicit `num_*` parameters to override

### Impact

- **API confusion**: Unclear which path is preferred
- **Precedence ambiguity**: What if both are passed with conflicts?
- **Documentation burden**: Must explain both paths

---

## Recommended Fix

Either:

1. **Remove explicit params**, require slot_config:
```python
def __init__(self, obs_dim: int, slot_config: SlotConfig, ...):
    num_slots = slot_config.max_slots
    # No override path
```

2. **Factory methods** for different use cases:
```python
@classmethod
def from_slot_config(cls, obs_dim: int, slot_config: SlotConfig, ...) -> Self:
    return cls(obs_dim, num_slots=slot_config.max_slots, ...)

@classmethod
def from_dims(cls, obs_dim: int, num_slots: int, num_blueprints: int, ...) -> Self:
    return cls(obs_dim, num_slots=num_slots, ...)
```

3. **Document precedence** clearly:
```python
"""
Args:
    slot_config: Primary configuration source (recommended)
    num_slots: Override slot_config.max_slots if both provided
    ...
"""
```
