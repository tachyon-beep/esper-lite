## Detailed Description

### What's Wrong

```python
def _emit(self, event: TelemetryEvent) -> None:
    event.env_id = self.env_id  # type: ignore[attr-defined]
    event.device = self.device  # type: ignore[attr-defined]
```

The `_emit()` method directly mutates the event object's attributes. Issues:

1. If `TelemetryEvent` is a frozen dataclass, this will raise `FrozenInstanceError` at runtime
2. The `# type: ignore` comments suggest the type checker already flags this
3. Inconsistent with `emit_with_env_context()` (L473-503) which correctly uses `dataclasses.replace()`
4. Mutating the input could cause unexpected behavior if the event is reused

---

## Recommended Fix

Use `dataclasses.replace()` to create a new event with injected fields:

```python
def _emit(self, event: TelemetryEvent) -> None:
    enriched_event = dataclasses.replace(
        event,
        env_id=self.env_id,
        device=self.device,
    )
    self.hub.emit(enriched_event)
```

This is consistent with `emit_with_env_context()` pattern already used elsewhere.
