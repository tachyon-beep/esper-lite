## Detailed Description

### What's Wrong

```python
# Line 390-394 (inside _validate(), called from __post_init__)
from esper.leyline.slot_id import validate_slot_ids, SlotIdError
try:
    validate_slot_ids(list(self.slots))
except SlotIdError as e:
    raise ValueError(f"Invalid slot configuration: {e}") from e
```

While Python caches imports, the lookup still has overhead. If TrainingConfig is instantiated frequently (e.g., in hyperparameter sweeps), this adds up.

### Impact

- **Micro-overhead**: Import lookup on every config creation
- **Profiling noise**: Shows up in tight loops
- **Unnecessary**: Module is already loaded after first import

---

## Recommended Fix

Move import to module level:

```python
# At module level
from esper.leyline.slot_id import validate_slot_ids, SlotIdError

# In _validate():
try:
    validate_slot_ids(list(self.slots))
except SlotIdError as e:
    raise ValueError(f"Invalid slot configuration: {e}") from e
```

If there's a circular import concern, use TYPE_CHECKING guard for type hints only.
