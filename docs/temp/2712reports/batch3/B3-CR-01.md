## Detailed Description

### What's Wrong

The `SeedMetrics.from_dict()` uses `.get()` with defaults for optional fields. If `counterfactual_contribution` key exists but with wrong type, it passes silently.

### Code Evidence

```python
# /home/john/esper-lite/src/esper/kasmina/slot.py:361-365

# Optional fields - these can legitimately be None/missing
metrics.counterfactual_contribution = data.get("counterfactual_contribution")
metrics._prev_contribution = data.get("_prev_contribution")
metrics.contribution_velocity = data.get("contribution_velocity", 0.0)
```

### Why This Matters Per CLAUDE.md

The project explicitly prohibits defensive patterns that mask bugs. If a checkpoint is corrupted or has schema mismatches, the code should fail fast rather than silently use defaults.

---

## Recommended Fix

Add explicit type validation:

```python
if "counterfactual_contribution" in data:
    val = data["counterfactual_contribution"]
    if val is not None and not isinstance(val, (int, float)):
        raise TypeError(f"counterfactual_contribution must be float|None, got {type(val)}")
    metrics.counterfactual_contribution = val
```
