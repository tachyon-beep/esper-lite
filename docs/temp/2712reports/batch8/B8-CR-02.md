## Detailed Description

### What's Wrong

```python
# This pattern appears 20+ times in the main training loop:
slot = cast(SeedSlotProtocol, model.seed_slots[slot_id])
```

Each occurrence involves:
1. Dictionary lookup (`model.seed_slots[slot_id]`)
2. Runtime cast (no-op but adds overhead)
3. Creates a local variable that may not be cached

In a tight epoch loop, this redundancy adds up.

### Impact

- **Micro-overhead**: Each dict lookup has cost
- **Code duplication**: Same pattern repeated 20+ times
- **torch.compile risk**: Dynamic dict access inside loops can cause graph breaks

---

## Recommended Fix

Cache slot references at epoch start:

```python
# At start of epoch processing for each env:
slots_this_epoch: dict[str, SeedSlotProtocol] = {
    slot_id: cast(SeedSlotProtocol, model.seed_slots[slot_id])
    for slot_id in slots
}

# Then use cached reference:
slot = slots_this_epoch[slot_id]
```

Or add a typed property to model:

```python
@property
def typed_seed_slots(self) -> dict[str, SeedSlotProtocol]:
    return cast(dict[str, SeedSlotProtocol], self.seed_slots)
```
